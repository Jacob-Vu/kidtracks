# KidsTrack — Use Cases
_Generated: 2026-03-18 | Version based on codebase snapshot_

---

## 1. App Overview

**KidsTrack** is a bilingual (English / Vietnamese) family web app (PWA) that helps parents build consistent daily routines for children by combining task management, pocket-money rewards, gamification, and progress analytics. It bridges parent oversight and kid motivation through a shared, real-time family workspace.

---

## 2. User Types

| User Type | Description | Primary Interface |
|-----------|-------------|-------------------|
| **Parent / Guardian** | Sets tasks, reviews progress, manages rewards and finances | Dashboard, DailyView, Templates, Ledger, WeeklyReport |
| **Child (Kid)** | Completes daily tasks, tracks balance, earns badges | KidDashboard, KidProfile |
| **New Visitor** | Exploring the app before signing up | LandingPage |

## 3. Primary Use Cases

### UC-01: Daily Routine Management
**Who:** Parent
**Goal:** Assign, monitor, and finalize a child's daily tasks with rewards or penalties
**Pain Solved:** Manual tracking of chores is inconsistent and hard to follow up on; kids forget or dispute what was assigned
**Scenario:**
- Parent opens Daily View for a child
- Adds tasks manually or loads a saved routine (auto-load)
- Marks tasks complete/incomplete at end of day
- Sets reward/penalty amounts and finalizes the day
- Balance is automatically applied to the child's ledger

---

### UC-02: Kid Self-Tracking
**Who:** Child
**Goal:** See today's tasks, mark them done, and track earnings and achievements
**Pain Solved:** Kids have no visibility into expectations or progress; motivation is lost without feedback
**Scenario:**
- Kid logs into their own account
- Views task list for today with status indicators
- Completes tasks (gets celebration animation + sound)
- Sees current balance, streak, and earned badges
- Checks savings goal progress

---

### UC-03: Template-Based Task Assignment
**Who:** Parent
**Goal:** Quickly populate daily tasks from reusable templates instead of typing each day
**Pain Solved:** Repetitive daily setup is tedious; parents give up and stop using the app
**Scenario:**
- Parent creates templates (e.g., "Brush Teeth", "Read 20 Minutes") with bilingual descriptions
- Imports pre-built packs (Little Star, School Star, Home Helper, etc.)
- Selects templates from the picker when setting up daily tasks
- Templates persist and can be shared across multiple children

---

### UC-04: Pocket Money & Ledger Tracking
**Who:** Parent + Child
**Goal:** Manage virtual pocket money tied to task completion
**Pain Solved:** Parents give money inconsistently and have no audit trail; kids don't know what they earned or why
**Scenario:**
- Parent sets per-task reward amounts (preset: 10k / 20k / 50k VND)
- Finalized days auto-credit/debit the child's balance
- Both parent and kid can view detailed ledger (earnings vs. penalties)
- Parent can add manual transactions (bonuses, deductions)

---

### UC-05: Savings Goal Tracking
**Who:** Parent + Child
**Goal:** Give children a target to save towards (toy, outing, etc.)
**Pain Solved:** Kids spend all earned money immediately with no delayed-gratification practice
**Scenario:**
- Parent creates a goal (title, icon, target amount, due date)
- Child's balance is tracked against the goal
- Milestones are auto-unlocked as balance crosses thresholds
- Progress bar and milestone timeline are visible to the kid

---

### UC-06: Weekly Performance Review
**Who:** Parent
**Goal:** Get a snapshot of the family's routine performance for the week
**Pain Solved:** No visibility into patterns — which days are hard, which tasks always fail, which child needs more support
**Scenario:**
- Every Monday, a weekly report modal appears automatically (or parent navigates to `/report/weekly`)
- Report shows: family completion %, per-kid breakdown, task popularity, best/worst days
- Earnings per child summarized
- Shareable summary for motivation or communication

---

### UC-07: Gamification & Badge Unlocking
**Who:** Child
**Goal:** Stay motivated through streaks, leaderboards, and badge achievements
**Pain Solved:** Kids disengage quickly without visible progress markers and rewards
**Scenario:**
- Kid maintains a daily completion streak (tracked across consecutive 100% days)
- Streak milestones trigger badge unlocks (e.g., 7-day streak = "Perfect Week")
- Kid views badge gallery; new badges animate on unlock
- Family leaderboard shows rankings among siblings
- Birthday badges unlock automatically

---

### UC-08: Voice Input for Tasks & Journals
**Who:** Parent + Child
**Goal:** Input task titles, descriptions, or reflections using voice instead of typing
**Pain Solved:** Typing on mobile is slow; small children cannot type; bilingual input is hard on keyboards
**Scenario:**
- User clicks mic button in task form or journal field
- Speaks in English or Vietnamese
- Audio is transcribed via Web Speech API; falls back to Google Cloud STT
- Transcribed text inserted into input field
- Parent reviews and saves; child adds journal reflection

---

### UC-09: App Install (PWA)
**Who:** New and returning users on mobile/desktop
**Goal:** Install KidsTrack as a native-like app on their device
**Pain Solved:** Opening a web browser URL every time is friction; app feels less "real" than installed alternatives
**Scenario:**
- Install prompt appears on landing page or after login
- On Android/Chrome: native install dialog triggered
- On iOS: step-by-step "Add to Home Screen" guide shown
- Installed users get icon on home screen, offline support, full-screen mode

---

### UC-10: Parent Onboarding (New Family Setup)
**Who:** New parent
**Goal:** Get from zero to first task assigned in under 5 minutes
**Pain Solved:** Complex setup discourages new users from continuing after signup
**Scenario:**
- Parent signs up via Google/Apple/Facebook/Email or quick-start username
- Onboarding wizard guides: create family → add first child → pick a template pack → assign first daily task
- Kid receives login credentials
- Family is live and ready to use

---

### UC-11: Kid Account Setup (Child Login)
**Who:** Child (guided by parent)
**Goal:** Set up a kid's own login so they can use the app independently
**Pain Solved:** Kids sharing parent logins creates privacy issues and confusion about which data is whose
**Scenario:**
- Parent provides their email address (family lookup)
- Kid creates their own username + password
- Kid customizes avatar (24 emoji options) and display name
- Kid logs in to their own dashboard

---

### UC-12: Multilingual Experience
**Who:** Vietnamese-speaking families / Bilingual families
**Goal:** Use the entire app in Vietnamese
**Pain Solved:** English-only apps are inaccessible to non-English-speaking parents and children
**Scenario:**
- Language switcher on login and header toggles between EN / VI
- All UI, labels, and notifications are localized
- Task templates have bilingual descriptions (en/vi)
- Voice input recognizes both `en-US` and `vi-VN`
- Currency formatted as Vietnamese đ

---

### UC-13: User Feedback Submission
**Who:** Parent + Child
**Goal:** Submit bugs, ideas, or general feedback to the product team
**Pain Solved:** Issues go unreported when there is no easy in-app path
**Scenario:**
- Parent or kid opens feedback launcher (floating button)
- Selects type: Bug / Idea / Feedback
- Optionally records voice or types text
- Submits directly from app

## 5. Pain-to-Feature Mapping Summary

| Pain Point | Feature That Solves It |
|------------|------------------------|
| Kids forget / ignore chores | Daily task list with kid-facing dashboard |
| No accountability for task completion | Parent finalizes day; rewards/penalties auto-applied |
| Pocket money given inconsistently | Ledger with auto-credit on day finalization |
| Kids spend all money immediately | Savings goals with milestone tracking |
| Daily setup is repetitive | Task templates + auto-load routine |
| No visibility into weekly patterns | Weekly report with analytics and heatmaps |
| Kids lose motivation quickly | Streaks, badges, leaderboard, confetti celebrations |
| Typing on mobile is slow | Voice input (Web Speech + Google Cloud STT) |
| App feels like a website, not real | PWA install prompt (iOS + Android) |
| Language barrier (Vietnamese families) | Full EN/VI bilingual support |
| Complex onboarding drops users | Onboarding wizard + quick-start login |
| Children can't use parent account | Per-kid login with avatar, profile, separate dashboard |
